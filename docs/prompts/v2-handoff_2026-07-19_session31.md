- **#115 잔여 순서(이슈 코멘트=정본)**: ③→⑤→④. ④ 게이트=이벤트 기반(discussion 이슈·PR 0+표면 변경 후 완주 토론 2회)+RustSec 트립와이어+워커 라이브 왕복 필수+스키마 스냅샷 레포 커밋 선행.
- **메이저≠큰 작업**: toml·lindera는 소스 호환 무변경이었다. 잔여 3종(rusqlite 57지점·rmcp 매크로·tungstenite ws)이 실작업.
- **시한폭탄 판정 3교집합**: 고정 리터럴 × 실벽시계 삭제/필터의 트리거 경로 × 테이블 의존 단언(인메모리 단언 무관). 단일 파일 분류기는 크로스 파일을 놓침 - 적대 스캔 필수.
- **훅 수정은 장수 세션에 소급 안 됨**: 배포 이전 무장된 Monitor는 옛 명령으로 계속 돈다. 잔재=세션 재시작 또는 원격 task 자가 재무장. src 소유권 가드 때문에 정리 순서=poll 교체 후 deregister.
- **위임 본문은 프리뷰에서 잘린다**: 발신=요지 선두 160자 / 수신=claim 전 get_task 전문 확인(훅+a2a-usage 규약 배포됨). completed task는 의도적으로 result만 반환.
- **워커 보고도 오진 가능**: "도구가 안 준다"는 보고는 자기 프로브로 교차 검증 후 믿기(#136 실원인=head -1 절단·completed 대상 테스트).
- **파서 다중화 계약**: 같은 설정 파일을 읽는 파서(py·ps1·rs)는 정본 지정+의미론 정렬이 계약. 개별 "개선" 봇 제안은 기각 근거.
