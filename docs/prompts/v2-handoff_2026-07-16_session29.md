## 확정 결정·교훈 (재론 금지)

- **cargo-dist 실측 함정**: 재생성이 `working-directory`·주석을 통과시키지 않음 → build-setup의 cd는 run 안에. release.yml은 순수 autogenerated 유지(수정=build-setup.yml·dist-workspace.toml 경유 + `dist generate`).
- **dependabot↔autogenerated 충돌 구조**: release.yml 액션은 dist 소유 → first-party 4종(checkout·upload/download-artifact·setup-node) 의존성 ignore + 수동 범프 절차(ci.yml·build-setup 동기 + dist generate). **Rust 0.x는 dependabot semver 분류상 minor라 ignore(semver-major)로 브레이킹을 못 거름** → cargo는 lockfile-only가 정답.
- **codex 생존 신호 실측 확정(사용자 협업 프로브)**: app-server `loaded/list`는 "한 번이라도 로드됨"이지 클라이언트 수명과 무연동(TUI 닫아도 잔류 실측) → `--remote` 시작 습관으로도 오라클 불가. 정답 = **래퍼 마커**(threadId↔래퍼PID, 이슈 #119 구현). VS Code 기동(source:vscode)은 래퍼 미커버=시간창 폴백. 빈 세션은 첫 메시지 전 rollout 미생성이라 어디에도 안 보임(정상).
- **claude mcp add는 멱등 아님**(같은 scope·이름 재실행=exit 1, CLAUDE_CONFIG_DIR 샌드박스 실측) → get 확인 후 add. 신규 MCP는 열린 세션에 핫 리로드 안 됨(공식 문서) → 재시작 1회는 구조적, 숨기지 말고 명시. user scope가 안전(local은 git-root 오귀속 실측).
- **로컬 무토큰 = 명시적 계약**(주석 "동일 계약"·회귀 테스트·결정 기록 3중) - init이 이를 활용(P0-①)해 첫 성공에서 토큰 어휘 소멸.
- **검증 체인에 `set -o pipefail` 필수**: `cargo clippy | tail`이 실패를 가려 실무자 green 오보 - 중앙검증이 잡음(#121 too_many_arguments).
- **/clear·프로세스 재시작은 persistent Monitor를 유령화**: #118 마커 자가종료가 근본 수정, 구세대 잔재는 양 머신 소진 완료. 재기동 중 send_task는 "Unable to connect" 순단 - health 200 후 재시도.
- **다중 홉 위임 실증**: 총괄→mac-codex(조사, relay 주입)→보고→총괄→mac-claude(처리)→보고 체인이 그대로 성립(래퍼 배치에 사용). relay 왕복 E2E 재검증 겸.
- **README 정본**: 첫 화면=첫 성공 경로(init·node·재시작만), chat=부수 기능, 지도 문서=docs/prompts/readme-rewrite-guide_2026-07-15.md(§5 체크리스트로 자가 검증).
