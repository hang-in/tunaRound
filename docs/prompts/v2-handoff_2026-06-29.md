## ③ v2 착수점 (로드맵, 우선순위)

v2는 독립 서브시스템이 여럿이라 **먼저 brainstorming으로 스코프·분해·우선순위**를 잡고 시작 권장. 후보:

1. **idle watchdog(INV-4):** 동기 러너를 line-by-line read + 활동 타이머 + watchdog 스레드로(tunaFlow `claude.rs` 참고). v1에서 미룬 robustness 1순위.
2. **Redis 멀티세션 = git-tree 다중 브랜치:** tunaSalon `session_bus.rs` 포팅 + presence/snapshot **신규 구현**(실측상 tunaSalon도 미구현). **브랜치=세션** 통합. `StoredMessage.parent_id`가 기반.
3. **N좌석 로스터:** 역할×엔진 동적 구성, 로컬LLM(tunaLlama)·opencode 좌석. tunapi `build_participants_from_config` 청사진.
4. **에이전트 쓰기 지목(RunMode::Write 실제 행사) = 협업 코딩 시작**(공유 작업공간). v1은 읽기 전용만.
5. **리치 프론트:** ratatui(tunaSalon `render_chat` 포팅) / web 게이트웨이(헤드리스 경계 위, 언어 무관).

## ④ 검증 / 가드레일

- **검증:** `cargo test` / `cargo build`(경고 0) / `cargo clippy --all-targets`. 검증과 commit/push 분리.
- **개발 규율:** `docs/reference/development-guidelines.md`(TDD red->green, 선제 설계 5규칙, 품질 셀프리뷰, #5 한국어 마침표, #6 파일 헤더, #7 checklist/context-notes). 이 프로젝트 실험 적용 중.
- **위임:** 구현=Sonnet 서브에이전트, Opus 리뷰(subagent-driven). routing = `.tuna-docs/routing.json`.
