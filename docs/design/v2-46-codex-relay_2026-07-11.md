# v2-46: codex-relay - sup 정체성 폐기, 보이는 세션 thread 직주입 (T4.7)

> 정본 배경: [세션19 핸드오프 §2-4·§4](../prompts/v2-handoff_2026-07-11_session19.md) - 사용자 재확정 "codex 주입 = 로스터에 보이는 그 세션의 thread. sup이라는 정체성 자체가 불필요, 익명 배달 데몬만 남는다".

## 1. 문제 (v2-37 잔재)

- codex 수신 경로가 `{machine}-codex-sup`이라는 **별도 정체성**(poll 데몬 + 사설 글루 thread)으로 남아 있다. 대시보드 GoalForm에서 codex에 던지려면 "win-codex 주입"이라는 백엔드 인프라 카드를 골라야 하고, 정작 로스터에 보이는 codex 세션 카드는 무효 대상이다.
- 주입이 세션 thread가 아니라 글루 thread로 가서, 사용자가 관전 중인 세션에는 아무것도 안 보였다(mac은 .thread를 운영자 세션으로 수동 교체해 임시 우회 = task f3026e3f).
- 글루가 .cmd 핸들러(subprocess) 경유라 CRLF·인용 함정이 반복됐다.

## 2. 설계

### 2-1. 새 데몬 `tunaround codex-relay` (머신당 1, role=infra)

주기 루프(기본 15초).

1. **로컬 라이브 codex 세션 열거**: `presence_scan::enumerate_codex_live` 재사용(rollout 파일 = 스캐너와 같은 SoR이라 로스터와 자동 일치). codex 세션 uuid = threadId.
2. 세션별 `poll_tasks(uuid)` → 새 submitted task마다,
   - `claim_task(task_id, agent=uuid, runner="codex")` - **relay가 대리 claim**(중복 주입 방지 + 재기동 무손실 + runner 트레이스. seen 디듑은 보조).
   - **in-process 주입**: `codex_inject::run`을 `--thread uuid` 모드로 직접 호출(subprocess·.cmd 핸들러 폐지). 주입 텍스트 = task id + 본문 + "답을 내고 complete_task로 마감하라"(claim 절차 지시 삭제 - relay가 이미 claim).
   - 주입 실패(세션 소멸·resume 거부·타임아웃) → `fail_task(task_id, 사유)` - dispatcher가 lease 만료를 기다리지 않고 즉시 실패를 본다.
3. 자기 등록 `{machine}-codex-relay`(tags: `machine={m}, role=infra, purpose=codex-inject`) + heartbeat + 코어 재기동 재등록. purpose를 유지해 대시보드 "codex 주입" 도트·GoalForm 유효성 판정이 그대로 동작한다.

인자: `--core/--token/--machine/--codex-dir/--stale-mins/--interval`(presence-scan과 동일 env 폴백) + `--ws`(기본 ws://127.0.0.1:8790).

### 2-2. codex-inject `--thread <id>` 직지정 모드

- thread 영속 파일을 **읽지도 쓰지도 않는다**. 지정 thread를 resume만 하고, 실패 시 새 thread 자가치유 **없이** Err(엉뚱한 곳에 답이 생기는 것 방지 - relay가 fail_task로 전환).
- 기존 `--agent` 파일 모드는 불변(하위호환·수동 디버그용).

### 2-3. GoalForm 유효 대상 재정의 (복귀)

- **codex 세션 카드 = 유효 대상**, 단 그 머신에 `purpose=codex-inject` infra가 online일 때(no-consumer 방지 조건이 "자기 poll 없음"에서 "머신 relay 없음"으로 이동).
- infra 항목은 대상 목록에서 **제외**(백엔드가 대상으로 노출되던 것 정리). 도트 렌더는 불변.

### 2-4. 폐기·이관

- `{machine}-codex-sup` poll 데몬·`codex-sup-handle.cmd`·글루 .thread 파일 폐기. restart-win-mesh.ps1의 codex-sup 단계 → codex-relay로 교체.
- mac 이관 = A2A task로 위임(relay 기동 + sup 데몬 중지, .thread 파일은 --thread 모드가 안 읽으므로 잔존 무해).

## 3. 비범위 / 미해결

- **win 멀티클라이언트 타임아웃**(§4 미해결): 사용자 TUI가 `--remote`로 attach된 thread에 주입 시 win에서만 응답 대기 타임아웃(mac 성공). 구현 후 라이브로 재현·규명한다. 재현되면 원인(브로드캐스트/승인 라우팅/버전)을 분리 기록하고, 주입 자체는 attach 없는 세션에서 검증.
- thread 로테이션(W4)·mesh 기억화(v2-45)는 불변 백로그.

## 4. 검증 계획

- 단위: relay 순수부(태스크→주입 텍스트 조립, 세션 열거 필터), codex-inject --thread 모드 분기.
- 라이브(win): GoalForm에서 win-codex-home 세션 카드 선택 → task → relay claim → 보이는 세션에 주입·답변 → completed. 관전 TUI attach 상태 재현 테스트 별도.
