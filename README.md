# tunaRound

터미널에서 진행하는 **사용자 주도 N-에이전트 설계 토론** 도구입니다.

기능이나 서비스를 만들기 전에, Claude Code와 Codex에게 서로 다른 역할을 부여하고 같은 레포 위에서 설계 토론을 진행합니다. 사용자는 진행자이자 최종 결정자로 남고, 두 에이전트는 각자의 도구로 레포를 읽으며 의견을 냅니다. 토론이 끝나면 결론을 문서로 남겨 바로 구현 단계로 이어갈 수 있습니다.

## 무엇인가

* **사용자가 진행을 잡는다.** 사용자가 질문하거나 방향을 제시하면 두 에이전트가 응답합니다. 에이전트끼리 자동으로 끝까지 토론하지 않습니다.
* **역할을 나눈다.** 한 자리는 아키텍트, 한 자리는 리뷰어처럼 역할을 주면 같은 레포를 보더라도 서로 다른 관점이 나옵니다.
* **같은 레포를 직접 본다.** 두 에이전트는 같은 작업 디렉터리를 각자의 CLI 도구로 읽습니다. 별도 컨텍스트를 길게 복붙하지 않습니다.
* **결론을 문서로 남긴다.** 토론 결과를 설계 문서로 기록해, 사이드 스레드에서 나온 결론을 다시 옮기는 단절을 줄입니다.

## 왜

tunaRound는 새 에이전트 프레임워크라기보다, 이미 검증한 구성요소를 터미널 네이티브로 얇게 묶는 도구입니다.

* 토론 진행 구조는 tunapi의 roundtable 코어를 가져옵니다.
* 에이전트 CLI 실행, 스트림 파싱, 러너 견고화는 tunaFlow의 러너 경험을 가져옵니다.
* 멀티세션, 관찰, 재개 구조는 tunaSalon의 Redis 패턴을 참고하되 v2로 둡니다.
