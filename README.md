# tunaRound

터미널에서 **사용자가 운전하는 2-에이전트(Claude Code·Codex) 설계 토론** 도구. 어떤 기능이나 서비스를 만들기 전에, 역할을 부여한 두 AI 에이전트와 함께 같은 레포 위에서 토론하고, 결론을 결과 문서로 남겨 그대로 구현에 들어간다.

## 무엇인가

- **사람이 운전한다.** 사용자가 말하면 두 에이전트가 응답하고, 사용자가 다음 발언으로 방향을 잡는다. 에이전트끼리 자동으로 끝까지 토론하지 않는다.
- **역할을 부여한다.** 한 자리는 아키텍트, 한 자리는 리뷰어처럼 역할을 주면 같은 자료를 봐도 다른 시각이 나온다.
- **같은 레포 위에서 토론한다.** 두 에이전트가 같은 작업 디렉터리를 자기 도구로 직접 읽는다. 별도 컨텍스트 주입이 없다.
- **결론이 곧 결과물이다.** 토론 결론을 결과 문서로 자동 기록한다. 사이드 스레드에서 복붙하던 단절이 사라진다.

## 왜

기존 도구의 강점을 터미널 네이티브로 얇게 결합한다.

- 토론 진행(역할·순차 응답·follow-up)은 tunapi에서 검증된 roundtable 코어를 옮긴다.
- 에이전트 CLI 구동(스폰·스트림 파싱·견고화)은 tunaFlow 러너를 옮긴다.
- 멀티세션·관찰·재개(Redis)는 tunaSalon에서 가져오되 v2로 둔다.
