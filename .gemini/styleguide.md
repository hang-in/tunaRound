# 리뷰 스타일 가이드 (Gemini Code Assist)

> Gemini Code Assist는 언어 설정 필드가 없어, 이 styleguide로 리뷰 언어를 지정한다.

## 언어

- 모든 리뷰 코멘트와 요약은 반드시 **한국어 존댓말**로 작성한다.
- 코드 식별자, 파일 경로, 명령어, 로그, API 이름은 원문 그대로 둔다(번역하지 않는다).
- em-dash 대신 일반 대시(-)나 콜론(:)을 쓴다.

## 리뷰 초점

- 버그, 회귀, 경계 조건, 에러 처리, 테스트 공백을 우선 지적한다.
- 스타일 지적은 짧게, 큰 문제가 없으면 그렇다고 말하고 잔여 리스크를 짚는다.
